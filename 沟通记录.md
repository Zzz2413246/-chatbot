### 五、图片识别深度排查与优化

32. **"你先审查一下本项目，然后优化一下识别图片的功能，目前不能正常识别图片"**
    - 全面审查发现 6 个根因：
      1. `ChatRequest` 缺 `image_data` 字段，前端发到 `/api/chat/stream` 的图片被 pydantic 静默丢弃
      2. `chat` / `chat_stream` 保存用户消息时未带 `image_url`，`_build_messages` 取历史时无图片
      3. `recognize_image` 默认模型 `gpt-4o-mini`，但未配 OpenAI Key
      4. `/api/models` 只返回字符串，前端期望 `supports_image` 字段，自动切换视觉模型失效
      5. `get_provider_config` 把 DeepSeek Key 错配给 OpenAI 模型（`or self.api_key` fallback）
      6. `chat_image_json` 硬编码 `image/jpeg` MIME
    - **修复**：`ChatRequest` 新增 `image_data`；保存消息时写入 `image_url`（data URL）；默认视觉模型改 `qwen-vl-plus`；`/api/models` 增加 `models_detail`（含 `supports_image`）；移除 openai/custom 的 `api_key` fallback；按魔数推断 MIME（PNG/JPEG/GIF/WEBP）
    - **增强**：后端视觉模型安全回退——非视觉模型收到图片时自动回退到 `qwen-vl-plus`，即使前端模型列表过期也能用
    - 验证：32×32 红色 PNG 经 `qwen-vl-plus` 正确返回"这张图片是红色的"（流式与非流式均通过）

### 六、数学公式乱码（续）

34. **（贴出分块矩阵识别结果的乱码）"在识别数学时会出现乱码，麻烦优化并排查是不是还有其他的问题"**
    - **根因**：模型输出标准 LaTeX 分隔符 `\[ ... \]`（块级）和 `\( ... \)`（行内），矩阵行用 `\\` 分隔；但 `renderMarkdown` 只保护了 `$$...$$` / `$...$`，marked 把 `\[`、`\\`、`\(` 里的反斜杠当转义符吃掉，导致 `\[` → `[`、`\\` → `\`、`\(` → `(`
    - **修复**：`renderMarkdown` 在提取 `$$` / `$` 之前，先把 `\[...\]` → `$$...$$`、`\(...\)` → `$...$` 统一转换，原有占位符保护逻辑即可覆盖
    - **顺带排查并修复的其它问题**：
      - 重新加载会话历史时图片不显示：前端读 `m.image_data`，后端 `to_dict()` 返回 `image_url`，字段名不匹配 → 改为 `m.image_url`
      - `image_url` 列类型 `String(2048)` → `Text`：data URL 实测可达 1.4MB，SQLite 不强制长度所以未崩，但换库会截断
    - 前端版本号 `v=5` → `v=6`
