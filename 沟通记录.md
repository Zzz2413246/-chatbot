### 二、图片识别排查（核心 Bug）

21. **"机器人目前只能点击图片，发送后不能识别⚠️ Error code: 400 - unknown variant `image_url`... 是不是因为 deepseek 模型不能识别图片"**
    - DeepSeek 不支持图片，需切换模型

22. **"sk-ws-...（千问 API KEY）这是千问的 apiKEY 可以识别图片，你自动给我添加一下吧，然后需要识别图片时自动切换就行了"**
    - 配置 qwen-plus 模型，上传图片时自动切换

23. **"怎么现在又不能登录，也不能自行切换大模型了"**
    - 浏览器缓存旧 JS，加版本参数 `?v=2` 解决

24. **"你好呀～我是科科！... 还是不能识别图片"**
    - 切换到 qwen-plus 后仍无法识别

25. **"图片 识别图片... 还是说不能直接识别图片，快点排查错误原因"**
    - 非流式 `/api/chat` 未传 `image_data`，已修复
    - 但 qwen-plus 仍返回"我还没收到图片"，`prompt_tokens=152` 异常偏低

26. **"此时就是千问模型"**
    - **根本原因**：`qwen-plus` 是纯文本模型，不支持图片！需用 `qwen-vl-plus` 视觉模型
    - **修复**：`settings.py` 将 `qwen-plus` → `qwen-vl-plus`，验证通过（`prompt_tokens` 152 → 1403）

### 三、数学公式渲染修复

27. **（贴出分块矩阵的乱码输出）"出现了乱码输出"**
    - LaTeX 矩阵换行符 `\\` 被 `marked.js` 当作 markdown 转义吃掉
    - **修复**：引入 KaTeX，`renderMarkdown` 先提取 `$$...$$`/`$...$` 占位符再解析
    - 前端版本号 `v=4` → `v=5`
