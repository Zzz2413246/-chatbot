# LangChain 多轮会话项目 — 需求说明文档

> **版本**：v1.1（纯净版）  
> **日期**：2026-06-24  
> **项目定位**：基于 LangChain 的多轮会话系统，同时作为教学项目

---

## 一、项目概述

| 项目属性 | 说明 |
|----------|------|
| **项目名称** | `langchain-chat` |
| **项目定位** | 基于 LangChain 的多轮会话系统，同时作为教学项目 |
| **开发语言** | Python 3.10+ |
| **架构模式** | 分层架构（业务层 / 数据模型层 / 存储层 / 接口定义层 / UI 实现层） |
| **异步模式** | 全链路异步（`asyncio`） |
| **交互方式** | 当前 TUI（命令行），后期扩展 WebUI |
| **版本控制** | Git（每步 commit + tag，支持任意回退） |
| **环境管理** | uv（Python 虚拟环境 + 依赖管理） |
| **教学要求** | 每步遵循 3W1H 框架讲解，代码含详细中文注释，每步引导确认 |

## 三、完整功能清单

### A. 核心对话功能

| 编号 | 功能 | 详细要求 |
|------|------|----------|
| A1 | 多轮对话 | 基于 LangChain Memory 机制，保持完整上下文连贯性，支持连续多轮交互 |
| A2 | 流式输出 | LLM 响应采用逐 token 流式输出，使用 LangChain `astream` API，终端实时渲染 |
| A3 | 异步架构 | 全链路 `async/await`：LLM 调用、文件 IO、数据库读写、用户输入全部异步 |
| A4 | 可配置 LLM 后端 | 通用 OpenAI 兼容格式，通过 `.env` 配置 `API_BASE_URL`、`API_KEY`、`MODEL_NAME`，支持运行时切换不同模型提供商 |
| A5 | 模型切换（会话内） | 同一会话中可中途切换模型继续对话（如从便宜模型切换到强模型），切换时保留历史上下文 |

### B. 用户管理功能

| 编号 | 功能 | 详细要求 |
|------|------|----------|
| B1 | 创建用户 | 首次使用时创建用户，用户名唯一校验 |
| B2 | 切换用户 | 运行时可切换到其他已存在用户，自动加载该用户的数据 |
| B3 | 删除用户 | 删除用户及其所有关联数据（会话、消息、预设、配置），需二次确认 |
| B4 | 用户隔离 | 不同用户的数据（会话记录、预设、配置）完全隔离，互不可见 |

### C. 会话管理功能

| 编号 | 功能 | 详细要求 |
|------|------|----------|
| C1 | 新建会话 | 清空当前上下文，创建全新会话，自动生成唯一 session_id |
| C2 | 加载历史会话 | 浏览当前用户的会话列表，选择加载某个历史会话继续对话 |
| C3 | 会话列表 | 展示当前用户所有会话（标题、使用的模型、预设、创建时间、最后更新时间） |
| C4 | 会话重命名 | 修改会话标题（支持自动标题 + 手动修改） |
| C5 | 删除会话 | 删除指定历史会话及其所有消息，需二次确认 |
| C6 | 会话自动保存 | 每轮对话完成（LLM 回复结束）后自动持久化，程序异常退出不丢失数据 |
| C7 | 会话标题自动生成 | 首轮对话后取用户首条消息前 30 字符作为会话标题 |

### D. 预设 Prompt 功能

| 编号 | 功能 | 详细要求 |
|------|------|----------|
| D1 | 系统内置预设 | 内置多种角色预设（翻译助手、代码专家、创意写手、英语老师等），所有用户共享 |
| D2 | 自定义预设 | 用户可自行添加/编辑/删除个人预设，每个用户独立管理 |
| D3 | 预设选择 | 新建会话时选择使用某个预设角色，也可选择"不使用预设" |
| D4 | 预设管理菜单 | TUI 中提供预设列表查看、新增、编辑、删除的操作界面 |

### E. 搜索与统计功能

| 编号 | 功能 | 详细要求 |
|------|------|----------|
| E1 | 对话搜索 | 在当前用户的历史会话中按关键词搜索消息内容，展示匹配的消息及其所属会话 |
| E2 | Token 用量统计 | 每轮对话结束后显示本轮消耗的 token 数（prompt_tokens / completion_tokens），累计显示当前会话总用量 |

### F. 导出功能

| 编号 | 功能 | 详细要求 |
|------|------|----------|
| F1 | 对话导出为 Markdown | 将指定会话的完整对话记录导出为格式化的 Markdown 文件 |
| F2 | 导出存储路径 | 导出到 `data/users/{username}/exports/` 目录，文件名包含会话标题和日期 |

### G. 基础设施功能

| 编号 | 功能 | 详细要求 |
|------|------|----------|
| G1 | 超时与重试 | LLM 调用超时自动重试，在 config.yaml 中配置超时时间和最大重试次数 |
| G2 | 结构化日志 | JSON 格式日志，记录关键操作（用户创建、会话操作）和错误（LLM 调用失败、存储异常），配置文件 `config/logging.yaml` 独立管理 |
| G3 | 配置管理 | `.env` 管理敏感信息（API Key、数据库密码），`config.yaml` 管理全局配置（模型列表、默认设置、存储类型），`config/logging.yaml` 管理日志配置 |

### H. 后期预留功能（当前阶段只定义接口，不实现）

| 编号 | 功能 | 详细要求 |
|------|------|----------|
| H1 | WebUI 接口 | 定义统一的 UI 协议接口，TUI/WebUI 通过实现同一接口对接业务层，WebUI 后期实现 |
| H2 | 多模型并行对比 | 预留接口和数据结构，支持同一 prompt 同时发送给多个模型，对比展示输出结果（基础功能完成后必须实现） |
| H3 | 图文上传 | 预留多模态输入接口（图片、文件上传及解析） |
| H4 | 语音输入/输出 | 预留音频处理接口（语音转文字、文字转语音） |
| H5 | 工具调用 (Tool Calling) | 预留 Agent/Tool 扩展接口（函数调用、外部工具集成） |

## 四、存储层设计

### 4.1 存储架构

采用**可插拔存储后端**设计：

- **抽象基类** `StorageBackend`：定义统一接口规范
- **工厂模式** `StorageFactory`：根据 `config.yaml` 中 `STORAGE_TYPE` 配置创建对应后端实例
- **三个后端实现**：`SQLiteBackend`（默认）、`MySQLBackend`、`FileBackend`

### 4.2 切换方式

```yaml
# config.yaml
storage:
  type: sqlite              # sqlite | mysql | file
  sqlite:
    path: data/sqlite/app.db
  mysql:
    host: localhost
    port: 3306
    user: root
    password: ""             # 建议放 .env
    database: langchain_chat
```

### 4.3 数据实体设计

| 实体 | 表名 | 字段 | 说明 |
|------|------|------|------|
| **User** | `users` | `id`(PK), `username`(UNIQUE), `default_model`, `default_preset_id`(FK nullable), `created_at`, `updated_at` | 用户基本信息与偏好 |
| **Session** | `sessions` | `id`(PK), `user_id`(FK), `title`, `model_name`, `preset_id`(FK nullable), `total_prompt_tokens`, `total_completion_tokens`, `created_at`, `updated_at` | 会话索引 |
| **Message** | `messages` | `id`(PK), `session_id`(FK), `role`(human/ai/system), `content`(TEXT), `prompt_tokens`(INT), `completion_tokens`(INT), `created_at` | 单条消息 |
| **Preset** | `presets` | `id`(PK), `user_id`(FK nullable, NULL=系统内置), `name`, `description`, `system_prompt`(TEXT), `is_builtin`(BOOL), `created_at`, `updated_at` | 预设角色（全局共享 + 用户私有） |
| **UserConfig** | `user_configs` | `id`(PK), `user_id`(FK), `key`(VARCHAR), `value`(TEXT), `updated_at` | 用户偏好键值对 |

### 4.4 实体关系

```
User (1) ──────< Session (N)       一个用户拥有多个会话
Session (1) ───< Message (N)       一个会话包含多条消息
User (1) ──────< Preset (N)        一个用户拥有多个自定义预设
Preset (1) ────< Session (N)      一个预设可被多个会话使用（可选）
User (1) ──────< UserConfig (N)    一个用户拥有多条配置
```
